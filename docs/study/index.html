AI Agent 核心原理

所有 Agent 框架的本质都是一个 while 循环 — claude-code-java 将这个循环抽象为可嵌入任何 Java 应用的 AgentLoop

// AgentLoop.java — 核心 while 循环 while ("tool_calls".equals(stopReason)) { response = llm.chat(messages, tools); stopReason = response.finishReason(); tools.execute(response.toolCalls()); messages.appendResults(toolResults); listener.onToolEnd(toolResults); } // stop_reason="end_turn" → 输出最终回复 listener.onTextDelta(response.text());
STEP 0
判断循环条件
stop_reason == "tool_calls" 时继续循环，"end_turn" 时退出
STEP 1
调用 LLM API
携带 messages 历史 + tool definitions，发起 streaming 请求
STEP 2
执行工具调用
ToolProvider 执行 bash/read/write/edit 等，结果收集为 toolResults
STEP 3
追回结果 + 通知
tool results 追加到 messages，AgentEventListener 推送 SSE 事件

Context 压缩 — 三层管道

ContextCompactor 通过三层机制保证 LLM 上下文始终在可用范围内，同时保留完整历史链供 Web Playground 追溯

Layer 1: Micro Compact
每轮自动 · 替换旧 tool_result
3轮以前的 tool_result → "[used bash]"
Layer 2: Auto Compact
messages ≥ 40 时 · LLM 生成摘要
全量 → 1条摘要 + .transcripts/ 存档
Layer 3: Manual Compact
/compact 命令 · 随时触发
_transcript_file 链 → 逐层回溯历史

10 项核心能力

每项能力都是 AgentLoop 的能力扩展，通过 ToolProvider 接口注入，无框架依赖

🔄
TodoManager
Agent 自追踪任务状态。多步骤任务 pending → in_progress → completed，超过阈值自动注入 system-reminder 提醒。
todonag-reminder自追踪
🧠
ContextCompactor
三层压缩管道：Micro（每轮替换旧 tool_result）→ Auto（messages ≥ 40 LLM 摘要）→ Manual（/compact）。压缩历史通过 _transcript_file 链式追溯。
micro-compactauto-compacttranscript-chain
⚡
BackgroundRunner
线程池异步执行耗时任务，fire-and-forget。任务完成后自动注入 system-reminder 通知主循环，不阻塞对话。
bg_submitbg_statusbg_drain
💾
TaskStore
JSON 文件持久化任务 + 有向无环依赖图。依赖解锁自动广播，重启后状态恢复。Teammate 通过 task_poll 自主认领。
task_createtask_updatedependency-graph
🌿
WorktreeManager
Git Worktree 目录级隔离，每个特性分支独立工作区。wt_run 在隔离目录执行命令，主目录不受影响。
wt_createwt_rungit-isolation
🎯
SkillLoader
按需加载技能提示词。扫描 skills/ 目录，模型主动调用 load_skill 时注入 prompt，不预占上下文 token。
load_skill按需注入zero-context-cost
📨
MessageBus
JSONL 文件收件箱/发件箱，支持多 Agent 间异步通信。Lead 和 Teammate 通过 msg_send/msg_read 传递消息。
msg_sendmsg_readmulti-agent
🗂️
SessionStore
REST 会话持久化（.sessions/{id}.json）。Lead/Teammate 子会话分开存储，进程重启后完整恢复，Web Playground 可加载历史。
lead-session-tm-子会话持久化
🔒
SecurityUtils
危险命令黑名单（rm -rf /、sudo、shutdown 等）+ 路径穿越防护。所有 bash 执行和文件操作均受保护。
黑名单路径穿越防护超时截断
🤝
TeammateRunner
Agent Teams 核心。Lead spawn Teammate 后，每个 Teammate 在独立线程运行完整 LLM 循环，每次工具调用后实时持久化 session，Web Playground 可实时查看。
spawn_teammate独立LLM循环实时持久化

快速上手

三步启动，选择最适合你的交互方式

第一步：配置 API Key

# 编辑 claude-code-java-start/src/main/resources/claude.properties
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
OPENAI_BASE_URL=https://api.openai.com
OPENAI_MODEL_ID=gpt-4o
# 或通过环境变量覆盖（优先级最高）
export OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx

第二步：编译

mvn compile

第三步：选择启动方式

🌐 Web Playground（推荐）
💻 CLI REPL
🔌 REST API
# 启动 Web Playground + REST API
mvn exec:java -pl claude-code-java-start \
  -Dexec.mainClass="ai.claude.code.Application"

# 访问 http://localhost:8080
mvn exec:java -pl claude-code-java-start \
  -Dexec.mainClass="ai.claude.code.Application" \
  -Dspring.profiles.active=cli

# Slash 命令：
/help       # 显示所有命令
/tasks      # Todo + TaskStore 双视图
/compact    # 手动压缩上下文
/team       # 查看 Agent Teams 状态
/worktree   # 管理 Git Worktree
# 同步接口
curl -X POST http://localhost:8080/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"message": "帮我创建一个 Hello.java"}'

# SSE 流式接口
curl -N -X POST http://localhost:8080/api/chat/stream \
  -H 'Content-Type: application/json' \
  -d '{"message": "用快排实现 TopK"}'
