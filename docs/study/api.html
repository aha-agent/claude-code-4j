同步接口
阻塞等待 Agent 完整响应后返回，适合简单调用
POST /api/chat 发送消息给 Agent（阻塞直到回复完整） ▼
Request Body
字段类型说明

200 OK
字段类型说明
sessionIdstring会话 ID（首次时为自动生成 UUID）
replystringAgent 的完整回复文本

  "reply": "当前目录包含：pom.xml, README.md, src/ ..."
}

SSE 流式接口
实时推送，14 种事件类型，支持思考/工具/压缩/Teammate 全流程
POST /api/chat/stream SSE 流式对话（text/event-stream） ▼
Request Body（与同步接口相同）
{
  "sessionId": "可选",
  "message": "用户输入"
}
SSE 事件格式
# 每个事件两行：
event: event_type
data:  {"key":"value"}

# 结束时（无 data）：
event: done
data:  {}
SSE 事件流实时演示
POST /api/chat/stream — 实时事件流
等待开始...
全部事件类型（14 种）
事件名Payload 字段说明
会话标识
session_id { sessionId } 流开始时推送，用于客户端绑定会话
思考过程（Extended Thinking）
thinking_start { } 模型开始思考推理
thinking_text { text } 思考内容增量（可能多次）
thinking_end { ms } 思考结束，ms 为耗时毫秒
文本流式回复
text_delta { text } 文字回复增量，可多次，累积得完整回复
工具调用（主 Agent）
tool_start { toolName, toolCallId, input } 工具开始执行，input 为 JSON 字符串
tool_end { toolCallId, output } 工具成功完成，output 为结果字符串
tool_error { toolCallId, error } 工具执行失败
Teammate（Agent Teams）
team_tool_start { agentId, toolName } Teammate 开始执行某工具
team_tool_end { agentId, toolName } Teammate 工具执行完成
team_text { agentId, text } Teammate LLM 文字输出（摘要）
team_done { agentId } Teammate 完成全部工作，退出
上下文压缩
compact_done { summary, transcriptFile } 压缩完成；transcriptFile 为 basename，可通过 /api/transcripts 加载历史
完成 / 错误
done { } 整次对话完成，流关闭
error { message } 发生错误，流关闭
客户端接收示例
// JavaScript — fetch + ReadableStream 手动解析
const resp = await fetch('/api/chat/stream', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ sessionId, message })
});
const reader = resp.body.getReader();
const decoder = new TextDecoder();
let buf = '';

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  buf += decoder.decode(value, { stream: true });
  const lines = buf.split('\n');
  buf = lines.pop();  // 保留不完整的行
  let eventType = '';
  for (const line of lines) {
    if (line.startsWith('event:')) eventType = line.slice(6).trim();
    if (line.startsWith('data:')) {
      const payload = JSON.parse(line.slice(5).trim());
      handleSseEvent(eventType, payload);  // 业务处理
    }
  }
}

会话与历史接口
查询会话列表、消息历史、Teammate 子会话、压缩前 transcript
GET /api/sessions 获取所有 Lead 会话（含 Teammate 数量） ▼

仅返回 Lead 会话（文件名不含 -tm-），按最后更新时间倒序。

Response Array Item
字段类型说明
sessionIdstring会话 UUID
previewstring对话预览摘要（第一条用户消息截取）
updatedAtstring最后更新时间（ISO 格式）
teammateCountnumber参与的 Teammate 数量（0 表示无 Agent Teams）
[
  {
    "sessionId": "550e8400-...",
    "preview": "帮我分析这个项目的架构",
    "updatedAt": "2026-04-16T10:30:00",
    "teammateCount": 2
  }
]
GET /api/sessions/{sessionId}/messages 获取完整消息历史 ▼ 
{
  "sessionId": "550e8400-...",
  "messages": [
    { "role": "user", "content": "你好" },
    {
      "role": "assistant",
      "content": null,
      "tool_calls": [...],
      "_reasoning": "思考内容",
      "_reasoning_ms": 1234
    },
    { "role": "tool", "tool_call_id": "tc-xxx", "content": "结果", "_display_content": "完整内容" }
  ]
}

私有字段（_ 前缀）仅用于 UI 渲染，不会发给 LLM API。

GET /api/sessions/{sessionId}/teammates 列出该 Lead 下所有 Teammate 子会话 ▼ 
[
  { "name": "alice", "sessionId": "550e8400-...-tm-alice" },
  { "name": "bob",   "sessionId": "550e8400-...-tm-bob" }
]

子会话 ID 格式：{leadSessionId}-tm-{name}，可进一步调用 /messages 获取 Teammate 工作细节。

GET /api/transcripts/{filename} 加载压缩前历史（transcript 文件） ▼

读取 .transcripts/{filename}，返回压缩前完整消息数组。basename 校验防止路径穿越（filename 仅允许 [\\w\\-]+\\.json）。

# compact_done 事件携带 transcriptFile 字段
# 用它直接请求：
GET /api/transcripts/transcript_20260416_103045.json

# Response: 消息数组，与 /messages 格式相同
[ { "role": "user", ... }, ... ]

配置参考
claude.properties · 环境变量 · 命令行覆盖

配置文件路径：claude-code-java-start/src/main/resources/claude.properties
优先级：System.getenv() > claude.properties > 默认值

配置项默认值说明
OPENAI_API_KEY— ★ 必填。API 密钥，优先读取同名环境变量
OPENAI_BASE_URLhttps://api.openai.com OpenAI 协议服务地址，兼容各种云厂商
OPENAI_MODEL_IDgpt-4o 模型 ID，如 claude-opus-4-6、deepseek-chat
CLIENT_TYPEopenai openai 或 anthropic（选择请求协议）
DEBUG_PRINT_PAYLOADfalse 设为 true 打印完整 API 请求/响应报文 
# claude.properties 示例
OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_BASE_URL=https://api.anthropic.com
OPENAI_MODEL_ID=claude-opus-4-6
CLIENT_TYPE=anthropic
DEBUG_PRINT_PAYLOAD=false
